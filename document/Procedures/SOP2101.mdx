# Creation and Review of Standard Operating Procedures

**Compliance with this document is mandatory for all MINData partner organisations**

### Document Control

|    Date    | Version | Details     |  Modified By   |
| :--------: | :------ | :---------- | :------------: |
| 2019-12-18 | 0.1     | First Draft | Manuel Nielsen |

## Purpose

This procedure is to provide the method by which new procedures are developed, or existing procedures are to be expanded or reviewed, to address procedural gaps relating to the MINData partnership and / or RODE platform.

## Scope

This procedure applies to all standard operating procedures associated with the MINData Data Sharing Partnership.

## Process

![Diagram](https://raw.githubusercontent.com/ManuShno/rode-static/main/diagrams/Procedures/SOP2101.drawio.svg)

| Item                                | Detail                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| :---------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Gap Identified**                  | A procedural gap has been identified as part of the procedure review or raised by end users of the RODE platform. This is submitted via an online portal or via email to a member of the Governance Committee.                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| **Gap Acknowledged**                | The receipt of the procedural gap is tabled in the Governance Committee meeting and minuted as acknowledged.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| **Author Assigned**                 | The Governance Committee will assign an author to create / modify the procedure to address the identified procedural gap.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| **Procedure Drafted**               | The author will investigate if the procedural gap would require rescinding existing procedures. The procedure draft is created using the procedure template. This must include:<br/>• Procedure code - SOP & 2 digit calendar year of creation & zero-leading number of procedure being created that calendar year<br/>• Procedure name<br/>• Version number<br/>• Effective date- to be generated when final approval is received<br/>• Rescinded procedure, if previous procedures exist<br/>• Document control<br/>• Purpose<br/>• Scope<br/>• Process diagram<br/>• Process detail<br/>• Responsibilities - role based assignment<br/>• Related documents<br/>• Review requirement |
| **Create Initial Draft**            | If no existing procedures are impacted by the creation of a new procedure an initial draft is created.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| **Redraft Existing**                | If existing procedures are impacted existing procedures are incorporated into the draft of a new procedure. The new procedure must cover all functions of existing procedures and address the identified procedure gap.                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| **Draft Approved for Consultation** | The draft procedure is submitted for the review of the Governance Committee. This review can result in a required redraft or minuted approval of draft procedure. This approval allows the procedure to publication for partner consultation.                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| **Draft Published for Comment**     | The draft procedure is made available to the public online. The users of the RODE platform w ill be informed of a new procedure being made available for comment upon logging into the platform. The consultation period will be made available with the draft and all comments received within the publication period must be reviewed by the draft author. The publication period must not be less than 4 weeks.                                                                                                                                                                                                                                                                     |
| **Redraft of the Procedure**        | The procedure is redrafted by the author in consideration of all comments provided during the publication period. If a provided comment is not addressed by a redraft a log of the authors reason for disclusion must be made for the Governance Committee’s consideration.                                                                                                                                                                                                                                                                                                                                                                                                            |
| **Redraft Approved**                | The governance committee reviews the redrafted procedure and the disclusion log. If the procedure is minuted as approved an effective date is assigned. If not approved, the author must redraft the procedure addressing the issues raised by the Governance Committee.                                                                                                                                                                                                                                                                                                                                                                                                               |
| **Procedural Change Notification**  | The RODE end users will be notified upon their next session of the changes to the Standard Operating Procedures. This notification must include the effective date.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| **Rescind Existing**                | If an existing procedure is to be rescinded, the new procedure must note all rescinded procedures. The status of the procedure being rescinded must reflect this change.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| **New Procedure in Effect**         | The new Standard Operating Procedure is in effect from the logged effective date.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |

## Responsibilities

| Task                                    | Responsibility                                         |
| :-------------------------------------- | :----------------------------------------------------- |
| Gap identification                      | End users from partner organisations / periodic review |
| Gap acknowledgement                     | MINData governance committee                           |
| Author assigned                         | MINData governance committee                           |
| Create initial draft / redraft existing | Assigned author                                        |
| Draft approved for consultation         | MINData governance committee                           |
| Draft published for comment             | Assigned author                                        |
| Redraft of procedure                    | Assigned author                                        |
| Redraft approved                        | MINData governance committee                           |
| Procedural change notification          | MINData governance committee                           |
| Rescind existing                        | MINData governance committee                           |
| New procedure in effect                 | MINData governance committee                           |
